# セキュリティ・安全性ポリシー

YOHAKU は心に触れるアプリです。セキュリティと、ユーザーの心の安全性を両方、同じくらい大切にしています。

## 🔐 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は、GitHub の Issues には**書かずに**、
リポジトリオーナーへ直接ご連絡ください。

## 🌿 心の安全性に関する設計原則

### 1. AI は診断しない

AIのシステムプロンプトで、以下を明示的に禁止しています:

- 「〜症」「〜障害」「〜病」などの病名・症状名の使用
- 治療的アドバイス
- 「頑張って」「もっと」という励ましの言葉
- スピリチュアル・占い・非科学的な助言
- ユーザーの気持ちを否定・評価する発言

### 2. クライシス検知は多層防御

深刻なキーワードが含まれている場合、以下の3層で検知します:

| 層 | 役割 |
|---|---|
| フロントエンド（追加時） | `AddModal` でユーザーがメモを追加した瞬間に検知し、窓口を案内 |
| フロントエンド（分析時） | `AIPanel` で分析ボタンを押した瞬間、AIを呼ぶ前に検知 |
| サーバー側 | `worker/index.js` でも検知し、409ステータスで返す |

### 3. 窓口情報は常にローカル

クライシス窓口の連絡先は、ネット接続がなくても必ず表示できるよう、
フロントエンドのコード内にハードコードしています。

### 4. プライバシー

- ユーザーのメモは **localStorage のみ** に保存され、サーバーには永続化されません
- AI分析のときだけ、メモ内容が一時的にサーバーを経由して Anthropic に送られます
- サーバー側ではログに本文を記録しないよう実装しています（`console.error` は発生したエラーのみ）

## 🔒 APIキー管理

- フロントエンドには **絶対にAPIキーを書かない**
- Cloudflare Workers の Secret (`wrangler secret put`) を使用
- `.env.local`, `.dev.vars` は `.gitignore` 済み

## 🤝 コントリビュート時のお願い

機能追加のプルリクをいただくときは、以下を意識してください:

- ユーザーの気持ちを評価・ランク付けするUI（ポイント、レベル、ストリークなど）は避ける
- 「〜すべき」「〜しなさい」という命令調の文言は使わない
- カラーや表現は、現在のやさしいトーンと矛盾しないように
